# AgentHub 技术设计文档

## 1. 技术总览

系统由两个主要应用组成：

1. `agenthub-backend`
   FastAPI 单体后端，负责 API、数据库、运行时、任务、工作区、预览、部署和 WebSocket。
2. `agenthub-frontend`
   Vue 3 单页应用，负责会话、智能体管理、文件工作台和任务界面。

整体上是前后端分离的单体架构，后端内部按运行域和业务域拆分。

## 2. 后端结构

后端主要分为七层：

1. API 层
2. Service 层
3. Runtime 层
4. Model 层
5. Schema 层
6. 基础设施层
7. 实时通信层

运行时又分为：

- `agent_runtime`
- `manager_runtime`
- `bootstrap_runtime`
- `event_runtime`

1. `auth`

6. `group_tasks`

11. `project_code`
12. `workspaces`

14. `previews`

### 协作域

- `agent_instances`

- `workspaces`

任务节点同时维护执行状态和复核状态，是任务协作设计中的关键点。

## 5. 消息驱动架构

消息是系统的主要触发入口。

主链路：

1. 写入消息
2. 广播消息已接收
3. 创建并分发事件
4. 根据群组类型进入不同运行路径

群组类型对应的运行方式：

1. `bootstrap`
   进入引导运行时
2. `personal`
   自动触发对应 agent
3. `project`
   根据 mentions 触发 manager 或普通 agent

## 6. 三套 Runtime

### Agent Runtime

负责普通执行型智能体：

- 构建提示词
- 装配工具、技能、MCP
- 选择引擎
- 执行并记录 trace

### Manager Runtime

负责项目群管家：

- 读取项目上下文
- 规划任务
- 修改 DAG
- 分配节点
- 触发执行
- 复核结果

### Bootstrap Runtime

负责智能体初始化阶段的引导对话。

## 7. 工作区与交付

项目群自动绑定一个 `Workspace`。

系统围绕 Workspace 提供三类能力：

1. 快照
2. 命令执行
3. 预览与部署

执行机制基于 Docker：

- Execution：沙箱命令执行
- Preview：静态目录预览
- Deployment：基于 Dockerfile 构建并启动容器

## 8. 任务 DAG

任务系统的核心是 `Run + Node`。

已具备能力：

- DAG 校验
- 环检测
- 节点依赖管理
- 认领与分配
- 节点完成与失败
- 节点复核
- DAG Patch

当前任务系统不是简单待办，而是轻量工作流引擎。

## 9. 引擎抽象

单 agent 支持多种执行引擎：

1. `internal_llm`
2. `agentscope_react`
3. `acp:codex`
4. `acp:claude_code`

这让角色层和执行器层解耦。

## 10. 能力装配

智能体能力拆成三类：

1. Tool
2. Skill
3. MCP

分别对应：

1. 可执行动作
2. 工作方法
3. 外部接入能力

## 12. 当前技术特点

AgentHub 当前技术设计有四个明显特点：

1. 消息驱动
2. Runtime 分域
3. DAG 编排
4. 交付闭环

## 13. 当前技术风险

1. SQLite 更适合早期阶段
2. 事件处理依赖应用内异步任务
3. Docker 运行依赖宿主机环境
4. 文件系统路径模式对云化不够友好
5. 观测能力还不够统一

## 14. 建议方向

1. 数据库迁移到 PostgreSQL
2. 引入更可靠的事件队列
3. 增强 tracing 和 metrics
4. 抽象远程 workspace backend
5. 增强多环境部署能力

## 15. 总结

AgentHub 已经具备较完整的技术骨架：以群组协作为入口，以多运行时为执行核心，以 Workspace 和 Docker 为交付底座，以任务 DAG 为复杂任务组织方式。这套结构已经足以支撑团队级 AI 协作产品继续演进。
